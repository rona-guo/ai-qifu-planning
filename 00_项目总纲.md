## 二、项目背景

- **项目名称**：AI企服操作系统（Agent Studio v2.0）
- **核心目标**：构建"信任枢纽 + 诊断引擎 + 品控中心"三位一体的 AI 企服平台，服务中小企业
- **当前阶段**：五层知识模型重构 + 8步诊断法框架完善 + 人才库管理系统搭建
- **商业模式**：OPC 超级个体模式（模块化年费 + 私有化部署）
- **技术栈**：Agent Studio v2.0 / Flask + SQLite + 纯HTML/JS / 智谱AI GLM-4-flash
- **核心壁垒**：五层知识模型（主体→事件→角色→规则→异常）、诊断引擎、专家网络

## 五、关键术语表

| 术语 | 定义 |
|-----|------|
| **Service OS** | 企业服务操作系统，超级个体OPC的外挂大脑，包含获客/诊断/交付/品控四引擎 |
| **OPC** | One Person Company，超级个体公司——不靠招人、靠方法论和工具放大个人能力 |
| **出血点** | 中小企业业务流程中导致持续性损失的环节（信息/流程/经验/决策四种类型） |
| **专家网络** | 以个人为信任枢纽，按项目制分账的各领域顶级顾问协作网络 |
| **孵化园渠道** | B2B2B模式，通过孵化园/企服公司触达入驻企业 |
| **四锚点** | 合规（底线）+ 效率（内部管理效率，AI是落地工具）+ 增长（生存）+ 创新（未来） |
| **开源·节流** | 诊断顶层分流：开源=对外经营（收入侧），节流=内部管理（成本侧），对应财报收入/成本 |
| **螺旋推进** | 8步诊断流程的增强机制：每步可出阶段性判断+置信度，年度循环加深 |
| **置信度** | 🔴高置信（多方印证）/ 🟡中置信（方向对但缺数据）/ ⚪低置信（仅凭描述） |
| **诊断格调** | 不否认客户，视角差异而非对错，从"另一个视角补充"切入而非"纠正" |
