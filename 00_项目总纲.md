### 📦 制品版本管理流程

#### 制品命名规范
```
[制品名称]_v[主版本].[次版本].md
示例：诊断Agent架构_v2.1.md

主版本号：重大变更（如整体重构、新增核心模块）
次版本号：小幅修改（如修正措辞、补充细节）
```

#### 版本索引（单一事实来源）
所有制品的最新版本登记在 `制品/INDEX.md` 中。
AI 读取制品前，必须先查 INDEX.md 确定最新版本号，避免读到旧文件。

#### 制品更新流程
1. 修改制品内容 → 更新文件头部版本号
2. 更新 INDEX.md 中的版本号
3. git commit 时写明"制品名 v旧→v新：变更摘要"
