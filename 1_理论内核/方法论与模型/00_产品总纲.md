## 一、业务定位

**一句话**：「我 + 专家网络」做中小企业老板的 AI 落地策划师——全景诊断 → 落地策划 → 持续陪跑，帮老板在 AI 时代看清楚整个棋盘。

- **AI 的角色**：效率引擎（让我一个人做到传统团队的深度），不是产品标签
- **对外定位词**：AI 落地策划师（源自硅谷 FDE 概念的升级版）
- **核心模式**：我 + 专家网络（非 OPC，突出个人信任枢纽 + 专家协作网络）
- **竞争对手**：不抢 AI 实施公司的生意，抢传统咨询公司的生意——用 AI 武器把价格打下来
- **「四不」边界**：不是软件代理商 / 不是培训师 / 不是外包公司 / 不是一次性咨询 / 不是 AI 实施公司

## 二、产品线全景

> 产品线之间是「主线 + 场景实践」关系：主线是方法论体系，每个具体项目是方法论的一次场景验证。

| 产品线 | 类型 | 状态 | 说明 |
|--------|------|------|------|
| **AI 企服外脑** | 主线 | 持续迭代 | 方法论体系：诊断 → 策划 → 陪跑。客户定位、商业模式、交付流程的总框架 |
| **智巡·劳动关系 AI 治理** | 场景实践 | 参赛中 | 政府视角的劳动风险智能风控平台，数字专员 7 岗位全链路。AI+人力资源服务大赛赛道 4 |
| **OPC 申请** | 组织载体 | 材料准备中 | AI 驱动的中小企业企服「诊断 + 品控」一人公司，入驻武昌阿里滨江亲橙 AI OPC 社区 |
| **书项目** | 理论外化 | 素材整理中 | 《折腾》——AI 时代企业数字化转型的方法论外化输出 |
| **1人+多agent运营体系** | 筹备中 | 待启动 | 1人公司 + 多AI Agent 协作的运营方法论与系统搭建 |

> 后续新项目 = 主线方法论的一次新场景实践。新项目在 `5_项目交付/` 下建目录，配套 `00_项目总纲.md`。

## 三、核心壁垒

**四懂 × 诊断方法论 × AI 效率工具**

- **四懂**：懂业务 + 懂精益 + 懂数据 + 懂 AI
- **诊断方法论**：8 步诊断流程 + 螺旋推进 + 置信度分级
- **AI 效率工具**：Agent Studio / 五层知识模型 / 向量化检索 / 混合算力

### 核心能力公式

```
（懂业务 + 懂精益 + 懂数据 + 懂 AI）
    ×（诊断方法论 + 专家网络）
    × AI 效率工具
```

## 四、四层项目框架

> 整个业务按「从本质到执行」分为四层，不同层有不同受众和产出物。

| 层级 | 内容 | 回答的问题 | 受众 |
|------|------|-----------|------|
| **第一层** | 业务本质与定位 | 这个生意为什么成立？ | 所有人 |
| **第二层** | 方法论与工具支撑 | 凭什么能做到？ | 自己、合作方（验证第一层主张） |
| **第三层** | 客户定位与商业打法 | 卖给谁、怎么卖？ | 客户、市场 |
| **第四层** | 政策与资本 | 怎么拿外部资源？ | 政府、园区、投资人 |

## 五、跨项目术语表

> 以下术语在所有产品线中通用，项目专属术语见各项目总纲。

| 术语 | 定义 |
|-----|------|
| **外脑模式** | 「我 + 专家网络」协作模式：以娜娜为个人信任枢纽，按项目制分账的各领域顶级顾问协作网络。对外不称 OPC。 |
| **OPC** | One Person Company（内部术语，对外不使用）。在武汉的定义：20 人以下小团队，非字面意义的"一个人"。 |
| **出血点** | 中小企业业务流程中导致持续性损失的环节（信息 / 流程 / 经验 / 决策四种类型） |
| **专家网络** | 以个人为信任枢纽，按项目制分账的各领域顶级顾问协作网络 |
| **孵化园渠道** | B2B2B 模式，通过孵化园 / 企服公司触达入驻企业 |
| **四锚点** | 合规（底线）+ 效率（内部管理效率，AI 是落地工具）+ 增长（生存）+ 创新（未来） |
| **开源·节流** | 诊断顶层分流：开源 = 对外经营（收入侧），节流 = 内部管理（成本侧），对应财报收入 / 成本 |
| **螺旋推进** | 8 步诊断流程的增强机制：每步可出阶段性判断 + 置信度，年度循环加深 |
| **置信度** | 高置信（多方印证）/ 中置信（方向对但缺数据）/ 低置信（仅凭描述） |
| **诊断格调** | 不否认客户，视角差异而非对错，从"另一个视角补充"切入而非"纠正" |

## 六、叙事框架

**当前定案版本：v3.6（2026-06-29）**

- **我卖的是什么**：帮中小企业老板做他一个人做不了的判断（方向 / 方案 / 执行三类判断）
- **AI 的角色**：我的效率工具，不是产品标签；让我以一个人的成本做大企业参谋部的分析深度
- **落地小队模型**：前线诊断（娜娜）+ 专属技术团队 + 茶饭叙弹性专家网络

## 七、产品演进机制（多向闭环）

> 知识库不只是存储，是产品孵化器。沉淀的内容反过来驱动**所有层**的迭代和生发。

**核心逻辑**：1 理论 → 2 产品 → 3 工具 → 4 运营 → 5 交付 → 6 沉淀 → 同时辐射回 1/2/3/4

**AI 主动检查的反馈方向**：

| 方向 | 触发信号 | AI 建议动作 |
|------|---------|------------|
| 沉淀 → 理论 | 某方法论在项目中验证有效/失败 | 更新 `1_理论内核` 的方法论，修正壁垒 |
| 沉淀 → 产品 | 同类需求3次以上、新场景出现 | 生发新产品线，更新 `2_产品设计` |
| 沉淀 → 工具 | 工具被3个以上项目调用 | 优化并沉淀到 `3_工具资产`，建议产品化 |
| 沉淀 → 运营 | 获客策略效果数据、运营问题 | 调整 `4_业务运营` 的策略和体系 |

**触发时机**：每次项目复盘时 / 定期回顾知识库时 / 娜娜主动提问"有没有什么新模式"时

**流程**：AI 检查信号 → 主动向娜娜建议 → 娜娜确认 → 更新对应层级 → 新产品线/调整正式生效

**案例闭环**：项目完成 → 优秀部分提炼为客户案例（`4_业务运营/客户案例/`）→ 精华部分迭代或生发新产品（`2_产品设计`）

> 详细判断标准和操作指引见 `资产索引.md` 的"产品演进机制"章节。
