## 二、已确立的设计模式

### 模式一：超级个体 OPC 商业模式

- **适用场景**：个人品牌强、方法论体系完备、有专家人脉网络、但不想规模化招人的创业者
- **核心思想**：一个人 + 一套方法论 + 一个AI工具集 + 一个专家网络 = 能独立交付多领域企业服务
- **实现要点**：
  - BP 叙事以"我"为主语，不用"我们""平台"
  - Service OS 定位为"外挂大脑"而非"产品"
  - 增长靠知识复利（模板积累）而非人员扩张
  - 提高单价、精选客户、深耕而非广撒网
- **确立时间**：2026-06-11
- **相关制品**：企业服务操作系统_商业计划书_v3.md

### 模式二：专家网络分账模型

- **适用场景**：一个诊断师（全科）+ 多个领域专家（专科）协作交付
- **核心思想**：
  - **转诊模式**：纯领域问题转介，收取15-25%介绍费
  - **合作模式**：Agent方案框架+领域内容共同交付，40-60%对分
  - **主导模式**：Agent为核心领域专家仅"喂规则"，诊断师占60-70%
- **实现要点**：诊断师做全科诊断和方案设计，专家做专科治疗；客户只认诊断师
- **确立时间**：2026-06-11
- **相关制品**：企业服务操作系统_商业计划书_v3.md

### 模式三：Service OS 四引擎架构

- **适用场景**：需要结构化管理获客→诊断→交付→品控全流程的企业服务个体
- **核心思想**：
  - **获客引擎**：活动运营、内容分发、人脉转化
  - **诊断引擎**：标准化访谈框架、出血点计算模型、行业模板库
  - **交付引擎**：Agent Studio（AI系统设计）+ 各领域专家工具集
  - **品控引擎**：验收标准库、客户反馈系统、持续陪跑机制
- **实现要点**：四个引擎不一定全部系统化，先靠人工跑通再逐步工具化
- **确立时间**：2026-06-11
- **相关制品**：企业服务操作系统_商业计划书_v3.md

### 模式四：孵化园 B2B2B 渠道模式

- **适用场景**：通过孵化园/企业服务公司触达入驻企业，放大获客效率
- **核心思想**：
  - **驻园诊断**：园区采购年框 ¥3-8万/年
  - **推荐转化**：免费诊断（园区买单）+ 深度服务（企业买单）
  - **白标授权**：方法论授权给园区自有团队 ¥5-15万/年
- **实现要点**：一个园区50-100家企业，转化10%即5-10个项目
- **确立时间**：2026-06-11
- **相关制品**：企业服务操作系统_商业计划书_v3.md §2.3

### 模式五：AI对话式诊断方法论

- **适用场景**：中小企业诊断，客户需求模糊，信任未建立
- **核心思想**：
  - **顶层分流**：开源（对外经营/收入侧）·节流（内部管理/成本侧），一句话分流
  - **五模型按需调用**：画布（画清生意）→3169（拆解结构）→价值链（价值穿透）→BSC（因果追溯）→合规清单（底线检查）
  - **螺旋推进**：每步可出阶段性判断+置信度，年度循环加深
  - **格调原则**：不否认客户，视角差异而非对错
- **实现要点**：
  - AI开场白：「您好！我是娜娜的AI助理，和她一起了解您的企业，说不定能看到一些您自己没注意到的角度。」
  - 画布9模块为对话框架，AI像咨询师一样顺着客户说往深追
  - 双层存储：结构化层（Agent调用）+ 原文层（娜娜查询）
  - 支持语音输入
  - 置信度标注：🔴高置信/🟡中置信/⚪低置信
- **确立时间**：2026-06-08~06-10
- **相关制品**：AI企服外脑_Agent工具集设计文档_v2.0.md

## 三、模式之间的关联关系

```
OPC 超级个体（模式一）
    │
    ├── Service OS 四引擎（模式三）← 个体的"外挂大脑"
    │       │
    │       ├── 交付引擎 ──→ Agent Studio（AI系统设计工具）
    │       ├── 交付引擎 ──→ 领域专家工具集
    │       ├── 诊断引擎 ──→ AI对话式诊断（模式五）← 入口+中枢
    │       └── 获客引擎 ──→ 孵化园渠道（模式四）
    │
    └── 专家网络分账（模式二）← 个体的"外挂团队"
```

核心逻辑：模式一（OPC）是底座，模式三（Service OS）是骨架，模式二（专家分账）和模式四（孵化园）是两条腿走路。
