## 分层原则

**向上复用，向下沉淀。** 上层是跨项目复用的资产，下层是具体项目的交付物。项目结束后，有价值的经验回流到上层。

**六层逻辑链**：

```
1我想做什么 → 2做成什么产品 → 3用什么工具 → 4怎么拿到客户 → 5给客户做了什么 → 6学到了什么
理论内核     →  产品设计      →  工具资产   →  业务运营       →  项目交付        →  知识沉淀
```

**闭环反馈**：沉淀不是只回到某一层，而是同时向四个方向辐射——理论可迭代、产品可生发、工具可优化、运营可调整。

## 使用约定

### 对话协议

| 场景 | 娜娜怎么说 | AI 做什么 |
|------|-----------|----------|
| 聊已有项目 | 开头带项目名，如"劳动风控的方案..." | 读该项目 `00_项目总纲.md` → 恢复上下文 → 开始对话 |
| 聊全局/产品设计 | "产品层面..." 或没提具体项目 | 读 `00_产品总纲.md` |
| 提新想法 | 直接说想法，如"我想做一个1人+多agent运营体系" | 按归档决策树判断归哪层 → 跟娜娜确认 → 建目录+总纲 → 开始 |
| 拿不准放哪 | 直接问"这个放哪" | 按决策树帮娜娜判断 |

### 归档决策树

1. **跨项目复用的方法论/框架/模型？** → `1_理论内核/`
2. **理论转化为标准化服务/产品形态？**（定义卖什么、包含什么、怎么定价） → `2_产品设计/`
3. **可复用的代码/工具？** → `3_工具资产/`
4. **怎么拿到客户和项目？**（运营计划、获客活动、公司运营体系） → `4_业务运营/`
5. **有明确目标和交付物的具体项目？** → `5_项目交付/`（新建项目目录 + `00_项目总纲.md`）
6. **项目经验/复盘？** → `6_知识沉淀/`
7. **拿不准？** → 先放 `0_外部文件/`，确认后再归位

> **同一个内容可能跨层**（方法论在1，产品定义在2，运营计划在4，实施在5）— 拆开放。
> **拿不准分类时，先问娜娜，不要自行新建层级。** 可以考虑新建项目/子目录，但新建前必须跟娜娜确认。

### 产品演进机制（多向闭环反馈）

> 知识库不只是存储，是产品孵化器。沉淀的内容反过来驱动**所有层**的迭代和生发。

**核心逻辑**：沉淀 → 分析模式 → 同时辐射到理论/产品/工具/运营

| 反馈方向 | 触发信号 | AI 建议动作 |
|---------|---------|------------|
| 沉淀 → **理论** | 某方法论在项目中验证有效/失败 | 更新 `1_理论内核` 的方法论，修正产品总纲壁垒 |
| 沉淀 → **产品** | 同类需求出现3次以上、新场景出现 | 生发新产品线，更新 `2_产品设计` 和产品总纲 |
| 沉淀 → **工具** | 工具被3个以上项目调用、代码优化 | 优化并沉淀到 `3_工具资产`，建议产品化 |
| 沉淀 → **运营** | 获客策略效果数据、运营问题暴露 | 调整 `4_业务运营` 的策略和体系 |

**触发时机**：每次项目复盘时 / 定期回顾知识库时 / 娜娜主动提问"有没有什么新模式"时

**流程**：AI 检查信号 → 主动向娜娜建议 → 娜娜确认 → 更新对应层级 → 新产品线/调整正式生效

**案例闭环**：项目完成后，优秀部分自动提炼为客户案例 → 进入 `4_业务运营/客户案例/` → 精华部分迭代或生发新产品 → 回到 `2_产品设计`

### 文件管理约定

1. **对话上传文件先放 `0_外部文件`** — 再根据内容判断归入哪一层
2. **新文件直接写到知识库对应目录** — 不再散落在workspace或其他位置
3. **新文件同步登记索引** — 每次新增文件后更新本索引
4. **新项目从 `_项目模板` 复制** — 改项目名即可，保持 `00_项目总纲.md` + 六子目录结构
5. **每个项目固定结构** — `00_项目总纲.md` / 外部输入 / 生成产物 / 交付文件 / 项目管理 / 商务文件 / 复盘沉淀
6. **文件命名规范** — `日期_版本_文件名`（如 `20260710_v7_参赛提案.docx`）
7. **版本管理** — 生成产物只保留最近3个版本，更早的版本删除；定稿版放交付文件目录
8. **复盘沉淀** — 项目结束后，有价值的东西提炼到 `复盘沉淀`，再回流到 `6_知识沉淀`
9. **天勤文件夹暂不纳入** — 后续按需整合
10. **新建子目录需确认** — AI判断需要新建子目录时，先提示娜娜确认
11. **文件操作用移动** — 归档后原文件删除，保持单一来源
