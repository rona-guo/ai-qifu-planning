# 天遂税务师事务所AI数字化项目总纲

> 创建：2026-07-13
> 状态：诊断信息已收集，正在结构化输出+方案细化

## 一、项目定位

- **客户**：天遂税务师事务所（化名）
- **客户关系**：朋友关系，娜娜线下已做过诊断但未结构化输出
- **需求**：AI数字化系统建设，客户预算10万以内
- **当前状态**：有初版方案（82人天/16.4万），需基于诊断信息+方法论+行业特点深化重构
- **关键约束**：预算10万以内；娜娜希望有陪跑（参考FDE模式）；私有化部署是娜娜的判断

## 二、企业画像（诊断已知信息）

| 维度 | 信息 |
|------|------|
| 规模 | 20+人，95%以上为专业交付团队 |
| 组织 | 老板+2副总（专业审核+交付管理）+1市场+1行政+专业团队 |
| 主营 | 鉴证、税务报告、咨询、筹划 |
| 客户 | 上市公司、国央企、军工行业，口碑获客（政府朋友/老朋友介绍），客户数量不多 |
| 盈利模式 | 项目收费 |
| IT现状 | 无系统，数据在个人电脑，历史档案几乎未数字化 |
| AI使用 | 个人零星使用，老板用AI写PPT较多 |
| 数据安全 | 客户税务资料需保密，已明确提出 |

## 三、核心出血点

1. **市场化获客能力不足**——老板最关注，但又觉得行业特点不能做自媒体推广
2. **项目经验无法复用**——经验在个人脑中，无沉淀机制
3. **项目进度不透明**——微信群沟通，无系统
4. **重复性基础工作**——专业人员做低价值工作
5. **新人培养体系几乎没有**——知识传承靠师傅带
