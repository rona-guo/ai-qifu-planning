> 版本：V2（基于方案1完善）
> 日期：2026年7月
> 标签：私有化部署 | 明道云底座 | 企微统一入口 | AI赋能合规执业

## 一、项目背景与建设目标

### （一）项目背景

贵所作为服务大型稳定客户的税务师事务所，在数字化转型中面临以下核心挑战：

- 数据保密要求极高：大客户项目资料严禁外泄，现有分散存储模式存在安全隐患；
- 执业合规与留痕：行业监管对工作底稿、审批流程、项目档案留存有严格要求；
- 业务流程规范化不足：项目进度、报告审批、员工与行政管理仍依赖人工协调；
- AI应用场景已明确：需在确保合规前提下，提升方案撰写、报告生成与客户服务能力。

**补充背景**（基于线下沟通）：

在与贵所深入沟通后，我们还发现了几个更深层的挑战：

- **专业经验没有沉淀下来**：每个项目做下来积累的经验、踩过的坑、找到的解决办法，都在做项目的人脑子里。下次遇到类似的项目，还是从头来。老员工走了，经验也带走了。
- **项目进度老板看不到全貌**：现在靠微信群沟通，哪个项目到哪一步了、卡在哪里了，老板得一个一个问。
- **新人来了没人系统带**：没有培养体系，新人靠师傅带，师傅忙起来就顾不上。
- **获客方式单一**：专业能力很强，但外面的人看不到。老板想推广，又觉得税务师事务所不适合做自媒体。

### （二）建设理念

在与贵所沟通中，我们就AI的应用达成了以下共识，这也是本方案的核心理念：

**第一，AI确实能替代部分人工工作，但AI能力目前还有瑕疵，不能一下子跑太快。**

现阶段AI在文字生成、知识检索、数据分析等方面已经可以帮上忙，但在专业判断、复杂推理、合规把关等方面还不能完全替代人。所以我们提的是"AI数字化"——用AI辅助人，不是用AI替代人。系统做初稿、做检查提示，最终判断和签字还是注册税务师负责。

**第二，要实事求是地跟客户做项目。**

不画大饼、不堆概念。先从最痛的问题开始解决，让团队用起来、尝到甜头，再逐步扩展。一步一个脚印，不追求一步到位。

**第三，现在做的事情，同时也是为未来的AI大爆发做准备。**

AI能力会越来越强，这是确定的方向。现在虽然不能所有事都交给AI，但有两件事必须现在就开始做：

| 准备方向 | 现在做什么 | 未来AI能力上来后 |
|---------|-----------|----------------|
| **数据准备** | 把散落在个人电脑、纸质档案里的底稿、案例、经验整理成结构化的知识库和数据库 | AI可以直接用这些数据做更深层的事情——自动风险扫描、智能筹划建议、行业趋势分析 |
| **AI习惯准备** | 让团队从简单场景开始用AI（政策问答、周报生成），逐步习惯人机协作的工作方式 | 等AI能力真的爆发了，团队已经会用、习惯用，能快速跟上，不会被动 |

**这就是本方案的核心逻辑：用有限的预算解决当下最痛的问题，同时为未来打好数据和习惯的基础。**

### （三）建设目标

采用"明道云全覆盖 + AI精准赋能 + 私有化部署"的策略，分步推进：

- **项目建设（2-3个月）**：完成全部功能模块——基础管理（人事/行政/财务/客户）+ 项目管理 + 报告生成与审批（含AI审核岗）+ 知识库与AI客服（含Agent数字员工）+ 新员工培训 + 基础设施部署。六大AI场景全部落地，团队上手使用。
- **持续服务（项目完成后）**：系统上线只是开始。在使用过程中，团队会发现新的需求——有些可以AI解决，有些可以流程优化解决；历史底稿需要逐步数字化，知识库需要持续充实；AI能力也会随技术发展持续深化。这些不是一锤子买卖，是持续进化的过程。
- **后续（按需）**：在稳定运行基础上，根据实际需要扩展可选模块（入离调转、外出出差、项目档案管理等），届时视需求另行商议。

## 二、系统总体架构与部署方案

### （一）技术架构

系统采用分层架构，从访问层到数据层各归其位，逻辑闭环：

![系统技术架构](技术架构图.png)

**数据逻辑闭环**——从数据流入到沉淀回流，形成持续积累的闭环：

```
客户提交文件 → AI提取关键数据 → 人工核准 → 结构化入库
                                                    ↓
              经验复用 ← 项目档案归档 ← 报告生成 → AI审核岗 → 审批流
                ↓                                        ↓
         下次同类项目推荐                           知识库积累
```

**数据分级管理**：

| 级别 | 数据类型 | 访问控制 |
|------|---------|---------|
| L1 公开 | 公司公告、培训资料 | 全员可见 |
| L2 内部 | 项目进度、任务分配 | 项目成员可见 |
| L3 敏感 | 客户档案、工作底稿 | 按项目授权 |
| L4 核心 | 客户财务数据、鉴证底稿 | 严格授权+操作留痕 |

### （二）部署架构

采用私有化部署模式，部署在贵所指定的云服务器上，确保数据主权完全归属贵所：

- **访问层**：企业微信PC/移动端统一入口，免注册单点登录；
- **应用层**：明道云搭建人事、行政、客户、项目、知识库、审批流等核心模块；
- **数据层**：业务数据存明道云；文件附件根据规模选择——数据量小用明道云附件，数据量大挂载NAS；
- **安全层**：私有化部署，权限分级，操作全留痕，AI调用做数据脱敏。

### （三）NAS知识库方案（可选）

为保障大客户项目文件、历史底稿、政策文档的长期安全存储与高速访问：

- **NAS端配置**：在群晖NAS创建共享文件夹，开启NFS服务，设置云服务器IP读写权限；
- **服务器端挂载**：在云服务器创建挂载目录，通过NFS协议将NAS共享目录挂载至本地；
- **方案优势**：附件不占明道云空间、RAID冗余防丢失、支持异地访问、后期可迁移至MinIO对象存储。

### （四）AI能力路线

方案中的AI场景全部用明道云自带能力（工作流、模板填充、AI助手、知识库检索等）实现，快速见效。所有数据完全在贵所自己的服务器上，不经过任何外部网络。

后续如有需要，可按需接入外部大模型做能力升级——等数据基础和团队习惯都准备好了再考虑，不急着上。

### （五）数据安全保障

| 措施 | 说明 |
|------|------|
| 私有化部署 | 所有数据存储在贵所自有服务器，不经公网传输 |
| 数据分级 | L1-L4四级数据分级，按级别控制访问 |
| 权限管理 | 按角色/部门/项目控制数据访问范围 |
| 操作留痕 | 关键操作可追溯，满足执业监管要求 |
| AI数据隔离 | AI调用时只传问题和必要上下文，客户敏感数据脱敏处理 |

## 三、AI场景落地全景图

根据业务需求紧迫性，选择六大核心场景进行落地。以下在原沟通基础上，结合行业特点做了深化：

### 场景一：自动方案初稿生成（售前）

**原沟通内容**：读取客户画像+历史案例，AI自动生成方案框架与核心内容。

**深化**：这个能力不仅用于售前方案，还能帮你们产出"可转发的专业内容"——行业税务分析、政策解读、实务指南。现有客户收到觉得有用，转给同行朋友，就是帮你们获客。老板参加行业活动时，有专业内容做支撑，不是空手去。

### 场景二：自动生成报告（售中）

**原沟通内容**：基于明道云工作流+模板填充，自动生成标准报告。

**深化**：
- **客户文件AI数据提取**：客户提交大量文件（财务报表/凭证/合同/发票等），AI识别文件类型+提取关键数据（收入/成本/费用/资产等），结构化入库
- **人工核准**：AI提取的数据必须经专业人员核对确认，确认无误后才入库。AI辅助提取，人工把关质量
- **报告初稿生成**：核准后的数据+模板→自动生成报告初稿，专业人员在此基础上修改
- **数据校验**：AI自动检查数据逻辑（勾稽关系、异常波动），标注需要关注的点，减少人为遗漏
- **AI审核岗**：副总审核时，AI预审标注风险点，缩短审核时间

**完整流程**：
```
客户提交文件 → AI提取数据 → 人工核准 → 入库 → 模板生成初稿 → AI审核岗 → 人工三级审批
```

**重要边界**：AI只做数据提取、初稿生成和检查提示，最终判断和签字由注册税务师负责。

### 场景三：企微AI客服/政策问答（全周期）

**原沟通内容**：客户通过企微自然语言提问，AI检索知识库返回相关政策。

**深化**：
- **内部人员也能问**：不仅客户问，你们自己的专业人员随时可以问"这个行业的研发费用加计扣除有哪些注意事项"，系统从知识库找答案+关联历史案例
- **税法更新追踪**：新政策出来，系统自动抓取+AI解读要点+标注对现有客户的影响
- **相似案例推荐**：新项目启动时，自动匹配历史相似项目，推荐可参考的经验和底稿

### 场景四：新员工培训含AI陪练（全周期）

**原沟通内容**：培训课程+考试+AI随时解答流程与表单问题。

**深化**：
- **基于真实案例**：用你们做过的真实项目（脱敏后）作为案例库，新人可以问"这种情况下该怎么处理"
- **模拟场景练习**：AI出题模拟鉴证场景，新人自己做判断，做完看标准答案和注意事项
- 这解决的不只是培训问题，是知识传承问题——老师傅的经验不用等有空才能教

### 场景五：项目进度管理（售中）

**原沟通内容**：任务分解+甘特图+风险标签，实时掌控项目状态。

**深化**：
- **替代微信群沟通**：任务指派、进度更新、文件交接有迹可循，老板打开手机看到所有项目进展
- **经验自动归档**：项目完成后，AI辅助提取关键经验、注意事项，归入知识库。下次做类似的项目，系统直接推荐"上次做这个行业的客户，你们遇到过这些问题"

### 场景六：项目周报自动生成（售中）

**原沟通内容**：自动抓取任务进展+AI文本生成，减少80%撰写时间。

（保持不变，这个场景已经足够具体）

### 两个AI特色功能

**特色一：AI审核岗**

在不改变原有三级审核（项目经理→部门经理→合伙人）的基础上，增加一个AI审核环节：

```
报告提交 → AI审核岗（预审标风险点）→ 项目经理审核 → 部门经理审核 → 合伙人审批
```

- AI不替代人工审核，是在人工审核前先过一遍，标注需要关注的点
- 审核人员不用从头到尾逐行看，AI已经帮你标了重点
- 提高审核效率，减少遗漏风险

**特色二：Agent数字员工（企微AI客服）**

做一个与企微结合的AI客服，作为"数字员工"上岗：

- 客户通过企微提问，AI检索知识库返回政策答案
- 内部人员也能问——"这个行业的研发费用加计扣除怎么处理"
- 7×24小时在线，随时响应
- 有岗位、有职责，是团队的一员

## 六、客户需配合的工作

### （一）资料提供

为精准搭建符合贵所业务习惯的系统，需提供以下资料：

- 公司管理制度（人事、项目、财务、行政管理等方面）；
- 业务手册与业务指导书（各业务流程、操作规范）；
- 项目管理全套模板（立项申请书、任务分解表、项目周报、结项报告等）；
- 税务师事务所执业相关模板（工作底稿、鉴证报告、筹划方案等）；
- 政策法规、行业案例等知识文档。

### （二）人员配合

- 指定1-2名关键业务对接人，配合需求调研与确认；
- 安排各模块业务骨干参与UAT测试，验证功能是否符合实际场景；
- **安排副总或资深税务师参与知识库结构设计和报告模板定义**——系统质量取决于行业知识输入的深度；
- 安排行政人员配合历史档案整理和底稿扫描。

### （三）数据准备（为未来AI打基础）

| 准备事项 | 说明 | 为什么现在要做 |
