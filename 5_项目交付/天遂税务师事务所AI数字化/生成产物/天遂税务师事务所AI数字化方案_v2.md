# 天遂税务师事务所AI数字化建设方案

> 版本：v2（基于诊断结构化输出重构）
> 日期：2026-07-13
> 预算约束：10万以内（首期）+ 陪跑（持续）

---

## 一、诊断摘要

基于线下沟通和结构化诊断，天遂税务师事务所的核心画像如下：

| 维度 | 现状 |
|------|------|
| **团队** | 20+人，95%为专业交付团队，管理极轻（老板+2副总+1市场+1行政） |
| **业务** | 鉴证、税务报告、咨询、筹划，服务上市公司/国央企/军工企业 |
| **获客** | 口碑获客（政府朋友/老朋友介绍），客户数量不多但质量高 |
| **IT基础** | 几乎为零——无系统，数据在个人电脑，历史档案未数字化 |
| **AI使用** | 个人零星使用，老板本人用AI写PPT较多 |

**核心出血点**（按老板关注度排序）：

1. 市场化获客能力不足——专业能力强但无法被潜在客户感知
2. 项目经验无法复用——经验在个人脑中，做完即散
3. 项目进度不透明——微信群沟通，老板看不到全局
4. 重复性基础工作——专业人员时间消耗在低价值工作上
5. 新人培养体系几乎没有——知识传承靠师傅带

**诊断结论**：天遂的专业能力很强，但**能力没有被系统化沉淀和复用**。AI的价值不是替代专业判断，而是让专业经验从"个人脑中"变成"组织资产"，从"做完即散"变成"持续积累"。

## 二、行业特点与AI机会

### 税务师事务所的行业特殊性

税务师事务所不是普通的服务企业，有三个深层特点决定了AI落地的方向：

**特点一：专业判断是核心价值，不是流程**

税务鉴证、税务筹划的核心不是格式化文本，是专业判断——同一个客户的税务数据，有经验的税务师能看到风险点和筹划空间，新人看到的是一堆数字。

→ **AI的价值不在"自动生成报告"，而在"辅助专业判断"**——帮专业人员更快地找到信息、识别风险、关联案例，让判断更高效、更全面。

**特点二：知识管理是最大瓶颈**

税法几乎每月都在变，每个客户行业不同，每个项目都有独特的税务特点。但这些知识分散在每个人的脑中和个人电脑里，没有沉淀机制。新人来了要从头学，老员工走了经验带走。

→ **AI的价值在"让知识可沉淀、可检索、可传承"**——不是简单的文档管理，是构建专业知识引擎。

**特点三：获客靠专业影响力，不是流量**

老板觉得"行业特点不能做自媒体推广"——这个认知需要修正。不是做搞笑短视频，是产出**专业税务洞察内容**。客户选税务师事务所，看的是专业能力是否可信。专业内容产出=专业能力外化=口碑增强=获客能力提升。

→ **AI的价值在"辅助专业内容产出"**——基于团队经验，AI帮写初稿、整理案例、产出行业洞察，让老板和专业人员的能力被更多人看到。

### AI落地原则

基于以上行业特点，本方案的AI落地遵循三条原则：

1. **辅助判断，不替代判断**——AI帮找信息、做初稿、提示风险，最终判断由注册税务师负责
2. **先沉淀后外化**——先让内部经验沉淀成组织资产，再逐步外化为获客能力
3. **私有化保障合规**——客户税务数据高度敏感，私有化部署是底线

## 三、AI落地方向

基于诊断结果和行业特点，AI落地方向从原方案的"6个自动生成场景"深化为以下6个方向：

### 方向一：专业知识引擎

> 让税法知识、行业经验从"个人脑中"变成"组织资产"

**解决什么问题**：税法频繁变动靠自我学习、知识分散在个人脑中、新人上手慢

**做什么**：

| 功能 | 说明 |
|------|------|
| 税法更新追踪 | 新政策自动抓取+AI解读要点+标注对现有客户的影响 |
| 行业税务知识库 | 按行业分类整理税务特点、常见风险点、筹划空间（如制造业研发费用加计扣除、军工企业保密要求下的税务处理） |
| 智能问答 | 专业人员随时提问："A行业企业研发费用加计扣除有哪些注意事项？"AI基于知识库回答+关联历史案例 |
| 相似案例推荐 | 新项目启动时，自动匹配历史相似项目，推荐可参考的经验和底稿 |

**价值**：新人不用等师傅有空才能问；专业人员不用每次翻遍法规；老板的经验变成团队可用的资产。

### 方向二：报告智能辅助

> 让专业人员聚焦判断，不聚焦排版

**解决什么问题**：重复性基础工作消耗专业人力、报告质量依赖个人状态

**做什么**：

| 功能 | 说明 |
|------|------|
| 工作底稿导入 | 客户提供的财务数据/税务资料导入系统，AI自动识别结构 |
| 报告初稿生成 | 基于工作底稿，AI生成鉴证报告/咨询报告初稿，专业人员在此基础上修改 |
| 数据校验 | AI自动检查数据逻辑（如勾稽关系、异常波动），标注需要关注的点 |
| 审核辅助 | 副总审核时，AI预审标注风险点，缩短审核时间 |

**重要边界**：AI生成的初稿必须经专业人员审核确认。AI辅助的是效率和全面性，不替代注册税务师的签字判断。

**价值**：报告从"2天手写"缩短到"半天修改"；数据校验自动完成，减少人为遗漏；副总审核效率提升。

### 方向三：项目协作与经验沉淀

> 老板看到进度，做完即沉淀

**解决什么问题**：项目进度不透明（微信群沟通）、项目经验无法复用

**做什么**：

| 功能 | 说明 |
|------|------|
| 项目进度看板 | 每个项目状态、负责人、关键节点、风险预警一目了然 |
| 工作底稿管理 | 电子底稿在线管理，按项目归档，支持检索和复用 |
| 经验自动归档 | 项目完成后，AI辅助提取关键经验、注意事项，归入知识库 |
| 任务协同 | 替代微信群沟通，任务指派、进度更新、文件交接有迹可循 |

**价值**：老板打开手机看到所有项目进度；做完的项目自动变成知识资产；不再靠微信群追进度。

### 方向四：工作底稿数字化

> 数据资产化，为AI提供基础

**解决什么问题**：纸质+电子混合管理、历史档案未数字化、数据基础几乎为零

**做什么**：

| 功能 | 说明 |
|------|------|
| 纸质底稿OCR | 历史纸质档案扫描+AI识别，转成可检索的电子文档 |
| 底稿结构化 | 按标准格式存储，关联项目、客户、年度、报告类型 |
| 权限管理 | 按角色控制访问权限，保障客户数据安全 |

**价值**：历史档案可检索可复用；合规可追溯；为专业知识引擎和报告辅助提供数据基础。

**说明**：此方向为渐进式推进，首期先搭建框架和规范，历史档案数字化在陪跑阶段逐步完成。

### 方向五：专业内容获客

> 不是做自媒体，是让专业能力被看见

**解决什么问题**：市场化获客能力不足、老板觉得行业特点不能做推广

**做什么**：

| 功能 | 说明 |
|------|------|
| 行业税务洞察 | 基于团队经验+AI辅助，产出行业税务分析、政策解读、实务指南 |
| 客户动态追踪 | AI追踪客户所在行业的税务动态，主动发现服务机会 |
| 专业内容管理 | 内容库管理，可按场合提取使用（客户拜访、行业活动、朋友圈分享） |

**认知修正**：这不是"做自媒体"，是**把团队的专业能力外化成可被感知的内容**。客户介绍客户时，有专业内容做背书，信任传递更高效。老板参加行业活动时，有专业内容做支撑，不是空手去。

**价值**：专业影响力提升→口碑增强→获客能力提升。

### 方向六：新人AI陪练

> 新人不靠等师傅，AI随时可问

**解决什么问题**：培养体系几乎没有、知识传承靠师傅带

**做什么**：

| 功能 | 说明 |
|------|------|
| 案例问答 | 基于历史真实项目，新人可以问"这种情况下该怎么处理" |
| 模拟场景 | AI出题模拟鉴证场景，新人练习判断 |
| 学习路径 | 根据新人岗位和进度，推荐学习内容 |

**价值**：新人成长周期缩短；老师傅的精力不被"教新人"消耗；知识传承不再依赖个人。

### 六个方向的关系

```
工作底稿数字化（数据基础）
        ↓ 支撑
专业知识引擎（知识沉淀）←→ 报告智能辅助（效率提升）
        ↓                    ↓
项目协作与经验沉淀（管理透明）
        ↓
专业内容获客（能力外化）  ←→  新人AI陪练（人才传承）
```

**先做底层（数据+知识+效率+管理），再做上层（获客+人才培养）**——这跟陪跑模式天然契合。

## 四、技术方案

### 架构概述

```
┌─────────────────────────────────────────────┐
│              统一门户（企微/钉钉集成）           │
├─────────────────────────────────────────────┤
│  知识引擎  │  报告辅助  │  项目协作  │  内容管理  │
├─────────────────────────────────────────────┤
│              AI能力层（大模型+私有化）           │
├─────────────────────────────────────────────┤
│          明道云（低代码平台+私有化部署）          │
├─────────────────────────────────────────────┤
│              数据层（工作底稿+知识库）           │
└─────────────────────────────────────────────┘
```

### 技术选型

| 层 | 选择 | 理由 |
|----|------|------|
| 应用层 | 明道云 | 低代码快速搭建+支持私有化+客户已心动 |
| AI能力 | 大模型API（私有化/本地部署） | 满足客户数据保密要求 |
| 数据层 | 明道云内置+文件存储 | 底稿、知识库、项目数据统一管理 |
| 集成层 | 企微/钉钉 | 团队日常沟通工具，零学习成本 |

### 数据安全方案

| 措施 | 说明 |
|------|------|
| 私有化部署 | 所有数据存储在客户自有服务器，不经公网传输 |
| 权限分级 | 按角色控制访问（老板/副总/项目经理/专业人员） |
| 操作审计 | 关键操作留痕，可追溯 |
| AI数据隔离 | AI调用时不传输客户敏感数据，只传必要的问题和上下文 |

## 七、为什么是这套方案

### 与原方案的核心差异

| 维度 | 原方案（v1） | 本方案（v2） |
|------|-------------|-------------|
| **出发点** | 从"要建什么系统"出发 | 从"诊断发现了什么出血点"出发 |
| **AI场景** | 6个浅层"自动生成"场景 | 6个深层方向，从数据基础到获客外化 |
| **核心逻辑** | AI替代人工做重复工作 | AI让专业经验沉淀为组织资产 |
| **预算** | 16.4万（超预算） | 10万首期+陪跑持续 |
| **陪跑** | 无 | 有，参考FDE模式 |
| **行业深度** | 通用AI应用 | 针对税务师事务所行业特点设计 |

### 核心价值主张

**不是给你建一套系统，是帮你把专业能力从"个人脑中"变成"组织资产"。**

- 你的专业人员很强，但能力没有被系统化沉淀
- 你的经验很值钱，但做完项目就散了
- 你的获客靠口碑，但专业能力无法被潜在客户感知

**AI在这里的价值是让这些"隐性能力"变成"显性资产"——可检索、可复用、可传承、可外化。这不是一次性交付的系统，是持续进化的组织能力建设。**
