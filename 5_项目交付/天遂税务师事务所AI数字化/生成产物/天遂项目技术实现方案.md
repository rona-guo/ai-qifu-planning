## 一、总体架构

```
┌──────────────────────────────────────────────┐
│        客户端（企微 / 钉钉 / 浏览器）            │
├──────────────────────────────────────────────┤
│        明道云（业务应用层 + 私有化部署）          │
│  ┌──────┬──────┬──────┬──────┬──────┐       │
│  │项目管理│知识库 │报告管理│内容管理│工作流 │       │
│  └──────┴──────┴──────┴──────┴──────┘       │
│         ↕ API集成                              │
├──────────────────────────────────────────────┤
│        AI能力服务层（外部 / 私有化）              │
│  ┌──────┬──────┬──────┬──────┐              │
│  │大模型API│向量检索│OCR  │数据抓取│              │
│  └──────┴──────┴──────┴──────┘              │
├──────────────────────────────────────────────┤
│        数据层（明道云内置存储）                   │
└──────────────────────────────────────────────┘
```

**核心思路**：明道云做业务层（表单、工作流、权限、项目管理），AI能力作为外部服务通过API集成进来。明道云是主干，AI是插件。

## 二、明道云能力边界

### 明道云能直接做的

| 能力 | 用于哪个功能 | 说明 |
|------|------------|------|
| 工作表/数据管理 | 所有功能 | 数据存储、分类、检索 |
| 看板视图/甘特图 | 项目管理 | 项目进度可视化 |
| 工作流/自动化 | 报告审批、经验归档 | 审批流程、定时触发、条件触发 |
| 文档管理 | 底稿管理、知识库 | 附件存储、分类、权限 |
| 角色权限 | 所有功能 | 按角色控制数据访问 |
| API集成 | 所有AI功能 | 对接外部大模型、OCR等 |
| 企微/钉钉集成 | 统一入口 | 消息通知、待办推送 |
| 私有化部署 | 数据安全 | 部署在客户服务器 |

### 明道云不能直接做的

| 需要什么 | 解决方案 | 集成方式 |
|---------|---------|---------|
| 大模型推理 | 接入外部大模型API | 明道云工作流→HTTP请求→大模型API→返回结果 |
| 语义检索 | 向量数据库 | 文档向量化→存入向量库→检索时语义匹配 |
| OCR识别 | 接入OCR服务 | 扫描件→OCR API→返回文字→存入明道云 |
| 网页抓取 | 税法更新追踪 | 定时脚本→抓取税务局网站→AI解读→存入明道云 |

## 三、各功能技术实现路径

### 功能1：税务知识库 + 智能问答

**技术可行性：✅ 完全可行（RAG架构，技术成熟）**

**实现路径**：

```
用户提问
   ↓
明道云工作流触发（API调用）
   ↓
向量检索：在知识库中找相关内容
   ↓
大模型：基于找到的内容 + 问题 → 生成回答
   ↓
返回明道云 → 展示给用户
```

| 组件 | 选型 | 说明 |
|------|------|------|
| 知识库存储 | 明道云工作表 | 按行业/税种/业务类型分类存储文档 |
| 向量化 | Embedding模型 | 把文档转成向量，存入向量数据库 |
| 向量数据库 | Chroma（开源）或 Milvus | 轻量级，可部署在同一服务器 |
| 大模型 | 见第四节选型建议 | 生成回答 |
| 税法追踪 | Python脚本+定时任务 | 抓取国家税务总局网站更新→大模型解读→存入知识库 |

**你需要做的**：
- 在明道云里建知识库工作表（分类、标签、文档附件）
- 把现有资料整理录入（客户内部人员可做）
- 配置工作流：用户提交问题→调用API→返回答案

**需要外部做的**：
- 向量化配置（把文档转成向量存入向量库）
- 大模型API对接
- Prompt设计（让大模型基于知识库回答，不要胡说）
- 税法抓取脚本

### 功能2：报告起草辅助

**技术可行性：✅ 可行（需设计好数据输入格式）**

**实现路径**：

```
专业人员导入底稿数据（明道云表单）
   ↓
选择报告类型 → 匹配模板
   ↓
工作流触发：底稿数据 + 模板 → 大模型生成初稿
   ↓
初稿存入明道云 → 专业人员修改
   ↓
提交审核 → 系统标注风险点 → 副总审核
```

| 组件 | 选型 | 说明 |
|------|------|------|
| 模板管理 | 明道云工作表 | 存储报告模板、结构、字段 |
| 数据输入 | 明道云表单 | 专业人员录入底稿数据 |
| 大模型 | 见第四节 | 根据底稿+模板生成初稿 |
| 数据校验 | 明道云工作流公式 + 大模型 | 规则校验（勾稽关系）+ AI校验（异常识别） |

**难点和对策**：

| 难点 | 对策 |
|------|------|
| 底稿数据格式不统一 | 首期先做1-2种最常见的报告类型（如汇算清缴鉴证），标准化输入格式 |
| 大模型不懂税务专业表达 | 设计专门的Prompt，注入税务报告写作规范和术语库 |
| 生成质量不稳定 | 专业人员必须审核修改，系统标注"AI生成，需人工确认" |

**你需要做的**：
- 定义1-2种报告的标准模板和字段（客户副总帮忙）
- 在明道云建表单和工作流
- 设计报告审批流程

### 功能3：项目管理 + 经验积累

**技术可行性：✅ 完全可行（明道云核心能力）**

**实现路径**：

```
项目立项（明道云项目表）
   ↓
任务分配 → 进度更新 → 文件上传（全在明道云里）
   ↓
老板看看板 → 一目了然
   ↓
项目完成 → 工作流触发 → 大模型提取关键经验
   ↓
经验存入知识库 → 下次类似项目自动推荐
```

| 组件 | 选型 | 说明 |
|------|------|------|
| 项目管理 | 明道云项目应用 | 看板视图、甘特图、任务管理 |
| 文件管理 | 明道云文档管理 | 按项目归档、权限控制 |
| 经验提取 | 大模型API | 项目完成后，AI总结关键经验 |
| 经验推荐 | 向量检索 | 新项目启动时，自动匹配历史相似项目 |

**这是明道云的主场，你熟悉明道云，可以自己搭。** 唯一需要外部的是经验提取的AI集成。

### 功能4：底稿电子化

**技术可行性：✅ 可行（OCR技术成熟）**

**实现路径**：

```
纸质底稿扫描 → 上传到明道云
   ↓
工作流触发：调用OCR服务 → 识别文字
   ↓
识别结果 + 原始文件 → 存入明道云
   ↓
按项目/客户/年度分类 → 可检索
```

| 组件 | 选型 | 说明 |
|------|------|------|
| OCR服务 | 百度OCR / 腾讯OCR / 阿里OCR | 按调用计费，约0.01-0.05元/次 |
| 存储 | 明道云 | 文件+识别结果存储 |
| 分类检索 | 明道云 | 按项目、客户、年度、报告类型分类 |

**首期只搭框架和规范**，历史档案批量扫描在持续服务阶段推进。客户内部行政人员可以做扫描工作。

### 功能5：专业内容产出

**技术可行性：✅ 可行**

**实现路径**：

```
主题选择（老板/副总定方向）
   ↓
素材整理（团队提供经验、案例、数据）
   ↓
大模型：素材 + 主题 → 生成文章初稿
   ↓
专业人员修改 → 定稿 → 存入内容库
   ↓
按场合使用（客户拜访/行业活动/分享）
```

| 组件 | 选型 | 说明 |
|------|------|------|
| 内容管理 | 明道云工作表 | 内容库，按主题、行业、场合分类 |
| 大模型 | 见第四节 | 根据素材生成文章初稿 |
| 客户动态追踪 | Python脚本 | 追踪客户所在行业税务动态 |

**难点不在技术，在内容**——需要老板或副总提供素材和专业判断，AI帮写初稿。

### 功能6：新人培训辅助

**技术可行性：✅ 可行（基于知识库扩展）**

**实现路径**：

```
整理常见业务场景（客户副总帮忙）
   ↓
场景存入系统 → 大模型生成练习题
   ↓
新人答题 → 系统给答案和解析
   ↓
也可以自由提问 → 基于案例库回答
```

复用知识库的问答能力，额外做场景库和出题功能。

## 四、大模型选型建议

客户要求私有化部署（数据保密），大模型方案有两种选择：

### 方案A：私有化部署开源大模型（推荐）

| 项目 | 说明 | 成本 |
|------|------|------|
| 模型 | ChatGLM4-9B / Qwen2.5-7B / DeepSeek-V2（量化版） | 免费（开源） |
| 服务器 | 需要GPU服务器（推荐RTX 4090或A10） | 一次性约2-3万，或云租赁约2000-3000元/月 |
| Embedding模型 | bge-large-zh（中文向量化模型） | 免费（开源） |
| 优点 | 数据完全在内网，不经过外部 | — |
| 缺点 | 需要服务器和运维 | — |

### 方案B：API服务 + 数据脱敏

| 项目 | 说明 | 成本 |
|------|------|------|
| 模型 | 智谱GLM API / 通义千问API | 按调用计费，约1000-3000元/月 |
| 数据安全 | 调用前脱敏（去掉客户名称、具体金额等敏感信息） | 需要做脱敏处理 |
| 优点 | 不需要服务器，成本低，模型质量更高 | — |
| 缺点 | 数据经过云端（虽脱敏），客户可能不接受 | — |

**建议**：先跟客户确认能否接受方案B（脱敏后API），如果能接受，成本更低、效果更好。如果不能接受，走方案A私有化部署。

## 八、技术风险和对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 大模型回答不准确 | 用户不信任系统 | Prompt设计+知识库约束（只基于知识库回答）+ 标注"AI生成需人工确认" |
| 底稿数据格式不统一 | 报告生成失败或质量差 | 首期只做1-2种报告类型，标准化输入 |
| 私有化模型性能不够 | 回答慢或质量差 | 先测试模型效果；不行就降级为API+脱敏方案 |
| 客户不投入行业知识 | 系统空壳，没人用 | 在合同中明确客户投入要求；知识库建设需要副总参与 |
| 明道云API限制 | AI集成不顺畅 | 提前测试明道云API能力；必要时用中间件（如Python服务做中转） |
| 用户接受度低 | 系统建了没人用 | 培训+试点+老板带头用+持续服务推动 |

## 九、你需要确认的事

1. **大模型方案**：私有化部署（需买服务器）还是API脱敏（成本低但数据过云端）？需要跟客户确认
2. **明道云私有化授权**：明道云私有化版需要什么授权费用？需要你确认
3. **AI技术外包人选**：你有没有合适的人？还是需要找？
4. **报告类型优先级**：首期先做哪种报告（汇算清缴鉴证？税务咨询报告？）？需要客户提供样本
5. **客户投入承诺**：客户是否愿意安排副总和行政人员参与？
