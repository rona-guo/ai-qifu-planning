## 一、你的企业可能正在"出血"

老板们都在问同一个问题：**AI这么火，我的企业该用什么、怎么用、值不值得做？**

但在回答之前，先看看你的企业是否存在以下情况：

| 出血点 | 典型表现 | 每年隐性损失 |
|--------|---------|------------|
| **信息流失** | 客户资料散落在员工微信、Excel里，人走数据走 | 客户流失、重复获客成本 |
| **流程断裂** | 同一份合同要走5个签字，全靠纸质流转 | 审批周期长、业务卡壳 |
| **经验断层** | 老员工的经验在脑子里，新人来了从头学起 | 培训成本高、出错率高 |
| **决策盲区** | 老板拍脑袋决策，缺乏数据支撑 | 决策失误、机会错过 |

**每一项都在持续吞噬你的利润。** AI不是万能药，但它是目前成本最低的止血工具——前提是用对方法。

## 三、技术方案：不神秘，很务实

> 这一节用大白话讲技术。可以跳过，但如果你想确认我们"不是PPT公司"，可以看看。

### 部署方式：私有化

```
你的服务器（内网）
┌─────────────────────────────┐
│  业务平台 + AI引擎 + 知识库   │
│  数据100%在你的机房里          │
│  我们远程维护，碰不到你的数据   │
└─────────────────────────────┘
```

**为什么不是SaaS（公有云）？** 因为你的合同、客户名单、财务数据上了公网就收不回来。这是不可逆的合规风险。

### 技术架构："1+2+3"模型

| 层 | 内容 | 作用 |
|----|------|------|
| **1个底座** | 业务中台（低代码APaaS平台，承载所有业务数据与应用） | 先有业务底座，再叠加AI——不是空中楼阁 |
| **2个引擎** | 规则引擎（确定性）+ AI引擎（概率性） | 规则引擎做"对就是对"的事，AI引擎做"大概率对"的事 |
| **3层防护** | 数据分级（L1-L4）+ 分级路由（公私混调）+ 人工复核（强制节点） | 数据安全→成本优化→人机协同 |

**关键设计哲学**：不追求100%无人化。AI承担机械重复劳动，人类把控质量和风险。"AI干活+人把关"是最容易被传统企业决策者接受的模式。

### AI怎么工作：人机协同四步闭环

```
第一步：规则引擎预审
  → 必填校验、格式检查、数值范围 → 过滤低级错误
        ↓
第二步：AI引擎 + 知识库
  → 语义分析、法规比对、风险标记 → 输出结构化结果（风险点+置信度）
        ↓
第三步：业务人员终审（强制，不可跳过）
  → 查看原文 + AI标记 + 引用来源 → 确认或驳回
        ↓
第四步：系统自动学习
  → 人工修正版本更新至知识库 → AI下次更准
```

**AI是参谋不是裁判。** AI干80%的活，人做20%的关键决策。

### 数据安全：四级分级

| 级别 | 什么数据 | 怎么处理 | 用什么模型 |
|------|---------|---------|-----------|
| L1 公开 | 宣传资料 | 可公开 | 任意模型 |
| L2 内部 | 周报、通知 | 内部流转 | 私有模型（建议） |
| L3 敏感 | 合同、客户信息 | 脱敏处理 | 私有模型（强制） |
| L4 核心 | 财报、商业机密 | 物理隔离 | 私有模型（内网隔离） |

**你的数据不会出现在任何公网上。**

### 平台选择：不绑定

我们不是某一家平台的代理商。根据你的情况选最合适的：

| 你的情况 | 推荐方案 | 年费参考 |

| 50-300人，需快速搭建 | 低代码APaaS平台 | 6,000-20,000元/年 |
| 已有钉钉/企微 | 在现有平台扩展 | 已有成本内 |
| 有特殊需求、预算充足 | 轻量定制开发 | 按需 |

**平台是你的资产，AI能力是我们的服务。我们不靠卖平台赚差价。**

## 四、项目评估：不是拍脑袋报价

> 在报价前，我们用"8要素模型"科学评估项目复杂度。这也是向客户证明"我们的报价是有依据的"。

| 维度 | 低成本/短周期 | 高成本/长周期 |
|------|-------------|-------------|
| 1. 数字化基础 | 流程相对清晰，有专人负责 | 依赖Excel/微信，职责混乱 |
| 2. 历史数据 | GB级，文档规范 | TB级，大量扫描件/非结构化数据 |
| 3. 私密性要求 | 内部数据，接受轻度脱敏 | 核心机密，必须物理隔离 |
| 4. 应用场景 | 纯文本处理（报告/合同） | 图文/音视频多模态处理 |
| 5. 决策链 | 老板直接拍板 | 多部门博弈，需求反复 |
| 6. 用户心态 | 拥抱变化，愿意尝试 | 恐惧AI，抵触新系统 |
| 7. IT能力 | 有兼职IT，能配合环境准备 | 无IT人员，环境需全托管 |
| 8. 合规要求 | 一般商业记录 | 强监管，需全量日志审计 |

**使用方式**：逐项打分（低=1分，高=3分），8-12分属于标准项目，13-18分属于复杂项目，19-24分属于重度定制项目。评分结果直接对应报价区间。

**Q：我们连OA都没有，能用AI吗？**
A：恰恰最需要。我们先帮你把业务数据理顺、流程跑通（建底座），再叠加AI能力。不是空中楼阁。

A：可以。最低3万跑起一个能力点。平台年费6千，服务器1万以内。先解决一个最痛的点，觉得靠谱再扩大。

**Q：AI会不会取代我们的人？**
A：不会。AI干机械重复的活（写初稿、查格式、找风险），人做关键决策。AI让你的团队从体力活中解放出来。

A：个人用没问题。但企业级要解决三件事：数据安全（自己搭数据上公网）、业务贴合（通用AI不懂你的业务）、持续运营（人走了Agent就没了）。我们解决的就是这三件事。

**Q：系统上线后谁管？**
A：我们提供年度运维。不用招IT，不用担心系统没人管。知识库我们帮你更新，Prompt我们帮你调优。也可以自己管，我们不绑死。

**Q：我们行业比较特殊，你们做过吗？**
A：AI的底层能力是通用的——不管什么行业，企业都有信息处理、知识应用、流程优化、决策支持的需求。不同行业的差异在于业务场景适配。我们做过的不一定覆盖你的行业，但方法论和标准打法可以快速适配。实在做不了的，我们直说，帮你链接合适的外部资源。
